# Telnyx Knowledge Base

A compiled **[LLM wiki](https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f)** assembled from Telnyx's public documentation surfaces — [support.telnyx.com](https://support.telnyx.com), [developers.telnyx.com](https://developers.telnyx.com), and related sources. Pages are Markdown with frontmatter that links back to the original articles. Any AI agent that needs grounded, citation-rich knowledge of Telnyx products can consume this corpus directly.

## Overview

Raw articles and reference material from public Telnyx sources are scraped and synthesized into topical Markdown pages by an internal Telnyx pipeline, then stored here as a queryable knowledge base. Every page cites its upstream sources via `sources:` frontmatter, so consumers can always trace a claim back to the authoritative article.

This repository is the **artifact layer** — purely Markdown + YAML, schema-defined, git-versioned. Any agent or indexer can consume it.

## Repository Structure

```
SCHEMA.md       # Canonical page + index format
wiki/           # Compiled LLM wiki
└── index.md    # Catalog and entry point for navigation
```

The internal organization of `wiki/` is determined by the ingestion pipeline and may evolve. Consumers should navigate via `wiki/index.md` rather than hard-coding paths.

## Schema and catalog

Because the wiki is consumed by downstream agents and tools, pages follow a defined format.

- **[`SCHEMA.md`](SCHEMA.md)** describes the expected page frontmatter, filenames, internal links, and the index catalog. Read it before adding or editing pages. The schema is currently a draft and will evolve.
- **[`wiki/index.md`](wiki/index.md)** is the top-level catalog and entry point for navigating the wiki. Every page is reachable from there. Treat it as a derived artifact, regenerated by the ingestion pipeline rather than hand-maintained.

## File format

All content files are **Markdown (`.md`)** and must conform to [`SCHEMA.md`](SCHEMA.md) — required frontmatter (including `sources:` URLs), link conventions, and filename rules. The schema is what makes the corpus reliably consumable; Markdown on its own is not enough.

## How content gets here

Content lands through an internal Telnyx ingestion pipeline that scrapes the upstream sources, synthesizes pages, and opens PRs into this repo. Hand-authored PRs are accepted for correcting synthesis errors, but they're the exception — the default surface for content changes is the upstream documentation itself.

If the wiki is wrong because the **upstream source** is wrong, fix it upstream (support.telnyx.com / developers.telnyx.com) and the next sync will reflect it. If the wiki is wrong because the **synthesis** is wrong, open an issue here and a Telnyx maintainer will route the fix to the ingestion pipeline.

All PRs — automated or hand-authored — require approval from a maintainer before merging.

## Consumers

This repo is designed to be consumed by AI agents that answer questions about Telnyx products. Because pages are deterministic Markdown with explicit citations, the same corpus can back agentic-retrieval pipelines, vector indexes, or any other consumption shape. Treat the wiki as a data dependency that may evolve; pin to a specific commit if you need short-term stability.

## Local setup

```bash
git clone git@github.com:team-telnyx/knowledge-base.git
cd knowledge-base
```

No build step — the repo is pure content.
